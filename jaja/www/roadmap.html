JaJa : Roadmap

The objective targetted of v1 is to provide a partial but robust support the ECMASCRIPT RFC for inclusion in other libraries such as widget factories, templating engines and client-side plug-in architectures. This initial version should be extensible and geared toward community participation.

Release 0.7.3: Various housekeeping

  • Re-validate code with JSLint
  • More tests for nested parens
  • More tests for nested arrays
  • More tests for complex combined expressions

Release 0.8.0:

  • Support for object literals

Release 1.0.0:

  • Adequate exception handling and error messages
  • Externalize the runner and compiler so that they can be optionnal
  • Tests with negative tests results for exceptions handling
  • More complete test suite and performance benchmarks
  • Cache the resulting function from compiled expression
  • Create an automated performance benchmark from the existing test suite
  • Ability to specify prohibited objects (by reference) such as document, eval, window, etc..

BACKLOG for v1.0 :

  • In the test suite, ability to run/focus on a single test
  • Setup an online feedback system
  • Ability to load a test from the test suite into the lab UI
  • Simple backend for saving tests to a DB (SimpleDB or App Engine)
  • Batch file for builds and minification
  • A generator of random javascript expressions for test purposes

BACKLOG for v2.0 :

The objective of v2 is to fully control the capabilities of a script while providing a near-perfect support for the ECMASCRIPT RFC

BACKLOG for v3.0 :

The objective of v2 is to provide a framework for compiler level optimization of the outputed code and the lambda tree. This should open the door to effective source code compression or optimizations such as seen in the google closure compiler or Microsoft Ajax optimizer.
